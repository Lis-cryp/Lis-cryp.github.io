Політика натисни, щоб розгорнути

Ключові пункти з Pinterest Developer Guidelines і Connect app — коротко, щоб одразу розуміти вимоги.

Розділ Що потрібно
1. Базові принципи Бути прозорими для користувача, працювати лише з авторизацією, не збирати логіни/паролі, не продавати дані, мати власну privacy policy.
2. Дозволено робити Інтеграції для реклами, контенту, креативу, e-commerce, вимірювання ефективності та робочих інструментів для брендів/креаторів.
3. Заборонено Автодії без згоди користувача, скрейпінг, обхід обмежень, фейкові інсайти, спам-інструменти, стимулювання неорганічної активності.
4. Публікація контенту Якщо показуємо контент Pinterest — має бути видиме посилання назад на Pinterest, без перекриття та без викривлення оригіналу.
5. Ads / аудиторії Потрібні чіткі повідомлення про використання даних, згода/opt-out, заборона використовувати дані Pinterest для таргету поза Pinterest.
6. Anti-abuse Продукт не має допомагати зловживанням. Pinterest може обмежити або вимкнути застосунок за порушення.
7. Connect app: старт Потрібен бізнес-акаунт, підтверджений email, прийняті Developer Terms, потім заявка на trial access у My apps.
8. Redirect URI Redirect URI має точно збігатися з URI в налаштуваннях застосунку. Без додаткових редіректів на третій URL.
9. OAuth 2.0 flow Authorization request → redirect handling → обмін code на access token → безпечне зберігання і оновлення токена.
